Conclusion¶

In this project, we conducted a comparative analysis of two machine learning algorithms — K-Nearest Neighbors (KNN) and Linear Support Vector Classifier (LinearSVC) — on a dataset consisting of 1300 samples, each with 25 numerical features and a corresponding label drawn from five possible classes. A separate test set, also consisting of 1300 unlabeled samples with the same feature structure, was provided for final prediction. The goal was to preprocess the data, tune each model appropriately, assess their performance, and ultimately choose the best-performing model for generating predictions on the test data.

The analysis began with loading and inspecting the training and test datasets. The training data contained an extra column (Unnamed: 0) used as an index and a label column for supervised learning. The test set contained an id column that was later dropped during preprocessing. Upon confirming that all 25 features were numerical and that there were no missing values, we proceeded to data preparation. We dropped the irrelevant index and ID columns and then applied standardization using StandardScaler to bring all features to the same scale — a crucial step for algorithms such as KNN and SVM which are sensitive to feature magnitudes.

Next, we split the training data into a training subset (80%) and a validation subset (20%) using stratified sampling to ensure class distributions were preserved. This setup allowed us to perform robust model evaluation without relying on the test set, which was strictly reserved for final predictions. Using GridSearchCV, we then tuned the hyperparameters for both algorithms. For KNN, we explored a range of values for n_neighbors from 1 to 20, as well as two weighting schemes: 'uniform' and 'distance'. The best performance was achieved with n_neighbors=18 and weights='distance', giving a cross-validation accuracy of approximately 70.9%. For LinearSVC, we experimented with different regularization strengths (C) and found that C=0.001 gave the best results, though with a lower cross-validation accuracy of around 55.9%.

We then evaluated both models on the validation set. KNN achieved a validation accuracy of 66.9%, showing strong precision and recall across most classes, particularly class 0 (precision = 0.84) and class 2 (recall = 0.73). In contrast, LinearSVC reached only 55% accuracy and showed weaker performance overall, particularly on classes 0 and 3. Given the superior performance of KNN in both cross-validation and validation evaluation, we selected it as the final model.

The chosen KNN model was retrained on the entire training dataset to maximize learning from the full sample size. We then used this final model to predict labels for the 1300 test samples. The output predictions were displayed directly in the notebook and were also saved in a format suitable for submission if required — a plain .txt file containing one predicted label per line, with no header or index. The first several predictions included values such as [3, 2, 4, 2, 1, 3, 4, 1, 4, 4, ...], consistent with the expected label structure of the task.

Through systematic preprocessing, parameter tuning, evaluation, and interpretation, we concluded that KNN with n_neighbors=18 and distance-based weighting was the most suitable algorithm for this classification task. It consistently outperformed LinearSVC and produced reliable predictions. This process illustrates the importance of comparative modeling, evidence-based selection, and principled machine learning practices in real-world data science workflows.
